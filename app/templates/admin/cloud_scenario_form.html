{{ icon("help", 16) }} Schema reference

Top level: key (unique slug), provider (yushacloud today; aws/azure/gcp reserved), name, account_id, region, description, plus the resource lists below. Saved scenarios are immediately usable by the cloud simulator (labs select them by key, or by a lab content node path: "scenario").

roles — {"slug", "name", "description", "permissions": ["service:action"]}. Services: compute, storage, iam, network, database, billing; * wildcards on either side; *:* = full administrator.

iam_users — {"username", "display", "roles": ["slug"], "expected_role", "mfa", "access_key_active", "last_used_days", "description"}. Plant findings: a non-admin expected_role holding an admin role = over-permissive IAM; an admin with last_used_days ≥ 90 = unused administrator.

password_policy — {"min_length", "require_numbers", "require_symbols", "mfa_required", "max_age_days"}. Below 12 chars or MFA off = weak-policy finding.

buckets — {"slug", "name", "public", "encrypted", "versioning", "intended_public", "description", "objects": [{"key", "size", "sensitive"}]}. public: true with intended_public: false = public-exposure finding.

vpcs — {"slug", "name", "cidr", "internet_gateway", "subnets": [{"slug", "cidr", "public", "description"}]}

security_groups — {"slug", "name", "description", "rules": [{"direction", "protocol", "port", "cidr", "description"}]}. Port 22 from 0.0.0.0/0 = open-SSH finding.

vms — {"slug", "name", "subnet", "security_group", "public_ip", "state", "size", "description"}. Subnet and security group must exist.

load_balancers — {"slug", "name", "scheme", "targets": ["vm-slug"], "listener", "description"}

databases — {"slug", "name", "engine", "subnet", "security_group", "publicly_accessible", "encrypted", "description"}. publicly_accessible: true OR a 5432-from-anywhere rule on its group = exposed-database finding.
