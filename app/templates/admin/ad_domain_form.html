{{ icon("help", 16) }} Schema reference

Top level: key (unique slug), name, netbios, description, plus the object lists below. Saved domains are immediately usable by the AD simulator (labs select them by key).

ous — {"slug", "name", "description"}

groups — {"slug", "name", "description", "builtin"}. Include domain-admins and domain-users for realistic scenarios.

users — {"sam", "display", "title", "ou", "groups": ["slug"], "enabled", "locked", "failed_attempts", "last_logon_days", "description"}. Plant findings: last_logon_days ≥ 90 = inactive account, locked: true = lockout incident, a low-privilege user in domain-admins = least-privilege audit.

computers — {"name", "ou", "os", "ip", "is_dc", "description"}. Exactly one DC recommended.

shares — {"slug", "name", "server", "path", "description", "acl": [{"group", "right"}]}, rights read | write | full. An over-broad domain-users entry creates a permission-audit finding.

gpos — {"slug", "name", "linked_to", "kind", "password_policy": {"min_length", "complexity", "max_age_days", "history"}, "lockout_policy": {"threshold", "duration_minutes", "window_minutes"}, "settings", "script"}. The first GPO defining a password policy is the effective one.
